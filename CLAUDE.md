## Архитектурные требования к коду (V3 Compliance)
1. **Async-First и Graceful Shutdown (CSP):** Все операции строго асинхронные с каскадным `CancellationToken`. Категорически запрещено жестко «убивать» рабочие потоки (.NET Thread Kill). Если макрос отменяется (глобальный Abort), а нода помечена флагом `IsCriticalSection`, ей выделяется Soft Timeout (5-10 сек) на безопасную остановку. При превышении времени система отменяет Token, принудительно разрывая I/O, и нода переходит в состояние ERROR с отписью в Black Box Log.
2. **Локализация:** Никакого хардкода строк. Все тексты хранятся в файлах ресурсов (.resx) для поддержки RU и EN.
3. **Телеметрия (Black Box Log):** Логи пишутся в формате NDJSON. Важно: для нод отправка логов выполняется СТРОГО асинхронно без ожидания ответа через in-memory буфер (`Channel<T>.TryWrite`). Блокирующие дисковые I/O операции в теле ноды (для логов) запрещены. Запись буфера на диск выполняет системный BackgroundService.
4. **Слабая связанность (DI):** Регистрация сервисов в DI-контейнере, внедрение через конструктор.
5. **Чистый MVVM:** Логика в ViewModels, управление через ICommand, пустой Code-Behind (*.xaml.cs).
6. **Оптимизация памяти (Zero-Allocation):** Минимизация аллокаций в куче (использование `ValueTask`, `Span/Memory`). Использование иммутабельных объектов (`record`, `FrozenDictionary`) для транзитных данных.
7. **Отказоустойчивость `system_map.json`:** Запись индекса системы идет строго по паттерну Atomic Replace (во временный `.tmp`, затем Replace оригинала). Система содержит «Диспетчер файлов», выполняющий Auto-Rebuild карты по полю `"Environment"` внутри JSON-макросов (при удалении или сбое `system_map.json`).
8. **Безопасность (Vault):** Любые пароли, токены шифруются строго через Windows DPAPI (Data Protection API). Хранение в открытом виде запрещено.
9. **Архитектура V3 (Unified Bus):** Топология работает на `Event-Data Bus`. Управление (когда делать) и данные (с чем делать) разделены. Передача данных — строго по композитным ключам `SessionID_DataID`.
10. **Оптимизация исполнения:** Приоритет отдаётся прямым WinAPI/WebSocket/HTTP вызовам.
11. **Изоляция OBS-нод:** Все ноды OBS Studio находятся строго в пространстве имён `ARK.UI.Core.Nodes.OBS` и в папке `Core/Nodes/OBS/`.
12. **Защита UI-потока:** Любой тяжелый парсинг/регулярные выражения — строго в `Task.Run`. В UI (через `Dispatcher`) возвращаются только готовые модели для рендеринга.
13. **Безопасность Clipboard:** Любые операции с буфером обмена (OLE) вызываются в главном STA-потоке (`Dispatcher.InvokeAsync`) с автоповторами (до 5 раз × 50 мс, код ошибки 0x800401D0).
14. **Жизненный цикл ноды (Node Lifecycle States):** При отладке и в runtime каждая нода обязана транслировать одно из 5 состояний: `IDLE`, `PROCESSING`, `WAITING`, `ERROR`, `ZOMBIE` (детекция через Heartbeat Ping/Pong).
15. **UI Connection Rules (Магнитное притяжение):** Реализовать механизм фильтрации портов в редакторе: при перетаскивании провода визуально активировать только те порты-приемники, которые соответствуют PortDataType (SIGNAL, TEXT, IMAGE, OBJECT, ANY) порта-источника. Несовместимые порты должны быть «невидимы» для текущего соединения.

## Node Event-Data Flow Standards (V3 Protocol)
В новой архитектуре V3 нет передачи полей через Reflection (`In:{ID}:{prop}`). Все ноды подчиняются стандартам «Шины».
- **Интерфейс портов (Унифицированный):** 
  - **Входы:** `Trigger In` (импульс старта), `Data In` (приём ID пакета с шины).
  - **Выходы:** `Success Trigger` (или логическое ветвление True/Passed), `Data Out` (передача DataID дальше), `Custom Data Out` (вывод фрагмента), `Error/Rejected Trigger` (маршрут сбоя), `Black Box Log` (асинхронная диагностика).
- **Fail-Fast Sessioning (Pre-flight check):** Любая нода при получении импульса в `Trigger In` проверяет `SessionID` из пакета управления. Если `SessionID` устаревший (отличается от текущего контекста), нода мгновенно отправляет импульс в `Error/Rejected Trigger` (защита от «наслоения» в многопотоке).
- **Шина данных (DataBus) и композитные ключи:** Хранение в шине контекста происходит по ключу `"{SessionId}_{DataId}"`. Если нода просит ключ по своему старому SessionId — шина прозрачно возвращает `null`, гарантируя изоляцию переменных при конкурентном вызове макроса.
- **Типизация портов (Connection Filtering):** На входе в порт `Data In` нода проверяет `PortDataType` (Signal, Text, Image, Object, Any). Если данные не поддерживаются логикой ноды, применяется правило «Прозрачности»: данные прозрачно уходят в `Data Out` без изменения для последующих нод.
- **StartNode (Инициатор):** Реализовать IsDeletable = false. При установке физического линка в OnConnected() мгновенно активировать downstream-узлы. При разрыве в OnDisconnected() принудительно возвращать зависимые узлы в IDLE.
- **Macro Reset:** Реализовать поддержку широковещательного события ResetRequest. При получении сигнала ноды обязаны выполнить сброс настроек из «Рабочего профиля» в «Дефолтный».

## Стандарты Хранения, Регионов и Очередей (Macro Policies)
- **Именование макросов:** Файлы сохраняются в формате `{UserDefinedName}_{ID}.json`. Находятся в `macros/beta/` или `macros/release/` (внутри JSON поле `"Environment"` указывает статус). Очереди — в `queues/` с авто-бэкапом в `queues/backup/`.
- **Ротация архива:** При понижении в "Beta" макрос уходит в архив `history/`. Оставляется не более 2-х старых версий макроса (остальные удаляются).
- **Многопоточность по умолчанию:** Макросы выполняются параллельно (`Concurrent`).
- **Строгая очередь (RegionID):** Если макрос добавлен в `RegionID` с настроенным `Priority` (1-999), планировщик переводит его в жестко последовательную очередь: 1 выполняется первым, за ним 2 и т.д. Если Priority=0, макрос вклинивается на общих условиях.
- **Context Binding:** Параметры запуска указываются в метаданных: `GLOBAL` (системно), `MANUAL` (путь к .exe), `FOCUS` (активация при получении окном фокуса через `title_filter`).

## Стандарты UI/UX (WPF)

### OBS Cascade Parameters Rule
- **Правило:** Параметры нод OBS загружаются по цепочке: Сцена → Источник → Фильтр. Реализация через `IObsCascadeNode` и динамические привязки `AvailableSources`/`AvailableFilters` к `SelectedScene`/`SelectedSource`. Хардкод имён в коде или в XAML запрещён.

### Marquee Scrolling Text Rule
- **Правило:** Все списки и выпадающие меню с динамическими именами (профили, устройства и пр.) обязаны использовать оборачивание текста: `<Grid ClipToBounds="True">` + `MarqueeTextBlockStyle` (активация `DoubleAnimation` на уход за пределы контейнера, расчет `ft.Width - tb.ActualWidth`).

### Node Parameters Focus Safety Rule (Anti-Crash on Delete)
- **ЗАПРЕЩЕНО:** Использовать `Storyboard` внутри `EventTrigger LostFocus/GotFocus` с `TargetName`. Удаление активной (сфокусированной) карточки с UI вызывает исключение (поиск удаленного элемента).
- **ПРАВИЛЬНО:** Анимации свечения фокуса реализуются СТРОГО через `<Trigger Property="IsKeyboardFocused" Value="True">` с локальными `Setter`. WPF самостоятельно сбрасывает Setter'ы при удалении элементов без крэша приложения.
